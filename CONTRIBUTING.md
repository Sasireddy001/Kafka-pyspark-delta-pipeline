# Contributing to Kafka → PySpark → Delta Lake Pipeline

Thanks for your interest in improving this project.

## How to Contribute

1. Fork the repository.
2. Create a feature branch: `git checkout -b feature/your-feature-name`.
3. Make your changes and add or update tests.
4. Run `make test` and `make lint` locally.
5. Open a pull request with a clear description and any relevant screenshots or architecture updates.

## Reporting Issues

Use [GitHub Issues](https://github.com/Sasireddy001/Kafka-pyspark-delta-pipeline/issues) to report bugs, request features, or ask questions.

## Code Style

- Follow PEP 8.
- Keep configuration environment-driven.
- Add `pytest` tests for new transformations or utilities.
- Update `ARCHITECTURE.md` and `SYSTEM_DESIGN.md` if the architecture changes.
